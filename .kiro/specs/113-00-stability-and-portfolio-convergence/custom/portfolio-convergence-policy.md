# Portfolio Convergence Policy

## 1. 目标

在多 Spec 并行推进下，保证执行节奏稳定、优先级透明、结果可收敛。

## 2. Spec 组合状态模型

- `active`: 当前周期内执行，必须可映射到北极星指标
- `deferred`: 暂缓执行，记录触发条件与恢复条件
- `archived`: 已完成或终止，保留可审计证据

## 3. 准入规则（进入 active）

一个 Spec 进入 `active`，至少满足：

1. 对应明确业务价值或风险收敛目标
2. 有可验证输出（代码、测试、报告或治理状态）
3. 不与更高优先级 Spec 存在未解冲突

## 4. 多 Spec 默认执行模式

当 `active` Spec 数量大于 1 时，默认执行模式：

1. 使用 orchestrate 进行批次并行编排
2. 每个 Spec 在执行后必须通过 gate 检查
3. 任一 Spec 出现高风险失败时，允许局部 stop，不影响其他批次

## 5. 排序规则

排序权重（由高到低）：

1. 业务价值（影响核心指标）
2. 风险等级（阻塞面与回滚复杂度）
3. 依赖关系（是否阻塞其他 Spec）
4. 交付紧迫性（里程碑窗口）

## 6. 收敛规则

- 连续两周期未达成最小增量产出的 active Spec，降级为 deferred
- 已完成且证据齐全的 Spec，转为 archived
- deferred Spec 重新进入 active 前，需重新通过准入规则
