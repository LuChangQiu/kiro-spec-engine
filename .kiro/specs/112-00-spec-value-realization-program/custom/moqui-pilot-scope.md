# Moqui 试点范围与基准链路（Task 4.1）

## 1. 试点目标

在 Moqui 主线场景下，验证 kse 主路径可执行并具备可观测输出：

`spec bootstrap → spec pipeline run → spec gate run → orchestrate`

## 2. 试点范围

- 试点 Spec（首批）：
  - `100-00-moqui-screen-endpoint-expansion`
  - `101-00-moqui-api-catalog-monitoring-bridge`
- 执行环境：`E:\workspace\kiro-spec-engine`
- 输出沉淀目录：`custom/pilot-evidence/`

## 3. 输入前置条件

- `.kiro/specs/<spec>/requirements.md` 存在
- Node 与 kse CLI 可用
- orchestrate 所需 agent backend 配置可用（若不可用，先完成 pipeline+gate 基线）

## 4. 基准链路命令（建议顺序）

### Step A：Bootstrap（如需新建试点 Spec）

```bash
kse spec bootstrap --name 113-00-moqui-pilot-sandbox --non-interactive --json
```

**预期输出：**
- `.kiro/specs/113-00-moqui-pilot-sandbox/{requirements,design,tasks}.md`

### Step B：Pipeline（单 Spec）

```bash
kse spec pipeline run --spec 100-00-moqui-screen-endpoint-expansion --json \
  --out .kiro/specs/112-00-spec-value-realization-program/custom/pilot-evidence/pipeline-100.json
```

**预期输出：**
- `.kiro/state/spec-pipeline/100-00-moqui-screen-endpoint-expansion/latest.json`
- `custom/pilot-evidence/pipeline-100.json`

### Step C：Gate（单 Spec）

```bash
kse spec gate run --spec 100-00-moqui-screen-endpoint-expansion --json \
  --out .kiro/specs/112-00-spec-value-realization-program/custom/pilot-evidence/gate-100.json
```

**预期输出：**
- `custom/pilot-evidence/gate-100.json`

### Step D：Orchestrate（多 Spec）

```bash
kse orchestrate run --specs "100-00-moqui-screen-endpoint-expansion,101-00-moqui-api-catalog-monitoring-bridge" \
  --max-parallel 2 --json
```

**预期输出：**
- `.kiro/config/orchestration-status.json`
- 控制台 JSON 结果（可重定向存档）

- 已明确每步的输入与预期输出路径
