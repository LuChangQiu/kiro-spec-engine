1) Why sce is different
---------------------------------------------------------------

sce is not just "AI prompt helpers". It gives you:

- Structured delivery: Requirements -> Design -> Tasks
- Multi-Spec orchestration: run multiple specs in parallel safely
- Measurable outcomes: KPI snapshot/baseline/trend for weekly review
- Built-in governance: lock/doc/env/workspace controls for teams
- Tool-agnostic usage: works with Claude/Cursor/Windsurf/Copilot/SCE


2) 90-second value proof
---------------------------------------------------------------

Run these commands in your project:

  sce adopt
  sce spec bootstrap --name 01-00-demo-feature --non-interactive
  sce value metrics sample --out ./kpi-input.json --json
  sce value metrics snapshot --input ./kpi-input.json --json

If command 4 succeeds, you get machine-readable KPI output with risk
signal and evidence paths.


3) Recommended workflow (single Spec)
---------------------------------------------------------------

  sce spec bootstrap --name 01-00-user-login --non-interactive
  sce spec pipeline run --spec 01-00-user-login
  sce spec gate run --spec 01-00-user-login --json


4) Recommended workflow (multi Spec / parallel)
---------------------------------------------------------------

  sce orchestrate run --specs "spec-a,spec-b,spec-c" --max-parallel 3
  sce orchestrate status
  sce orchestrate stop

Note: `sce spec bootstrap|pipeline run|gate run --specs ...` defaults
to orchestrate mode.


5) KPI observability quick commands
---------------------------------------------------------------

  sce value metrics sample --out ./kpi-input.json --json
  sce value metrics snapshot --input ./kpi-input.json --json
  sce value metrics baseline --from-history 3 --json
  sce value metrics trend --window 6 --json
