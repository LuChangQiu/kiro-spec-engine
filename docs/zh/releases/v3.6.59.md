# v3.6.59 发布说明

发布日期：2026-03-19

## 重点变化

- 发布了一个仅用于对齐的补丁版，使 npm 包、仓库 tag 与发布分支都指向同一份最终仓库状态。
- MagicBall 的 delivery、engineering 与 multi-project 契约能力与 `v3.6.58` 保持一致。
- 稳定了被 Git 跟踪的项目共享问题投影 aging 输出；发布/发版时不再因为分钟级 `age_days` 漂移把工作区写脏。

## 验证

- `npm run audit:release-docs`
- `npx jest tests/unit/problem/project-problem-projection.test.js --runInBand`
- `node scripts/git-managed-gate.js --fail-on-violation --json`
- 通过 `prepublishOnly` 执行 `npm publish --access public`

## 发布说明

- 这个补丁版仍然主要用于消除发版状态歧义，但也顺手修掉了发布链里最后一个治理噪音点：项目共享问题投影文件里的 `age_days` 现在按整天取值，不会再因为过了几分钟就触发 Git 脏工作区。如果你要求 npm、tag、GitHub 发布线完全一致，应以 `v3.6.59` 作为标准版本。
