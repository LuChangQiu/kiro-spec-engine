## 引言

这是一场跨越技术与哲学的对话。作为一名拥有 20 余年 IT 研发经验、2000+ 小时 AI 编程经验、长期心学实践者，我与 Grok 探讨了 AI 发展的核心趋势：**AI 正在从辅助工具转变为信息化世界的中心驱动力**。

这不仅是技术演进，更是一场范式革命——就像 AlphaGo 改变围棋界一样，AI 正在重塑整个 IT 世界。

## 核心观点

### 1. AI 发展的本质：从"被动服务"到"主动驱动"

**我的基本观点**：

当前 AI 发展趋势是**使用 AI 来驱动遗留系统的维护和迭代开发**，而非简单地给 AI 提供 MCP (Model Context Protocol) 来驱动原有系统的服务功能。

信息化世界的中心将会是 AI。具有各种驱动 AI 能力的引擎，将会很好地完成日常的开发、维护信息化系统的工作。原先以人类专家和响应模式推动的时代结束了。

这有些像围棋界的 AlphaGo 演进路数：
- IT 专家会借助 AI，极大提升自身的专业能力
- 但这个 IT 世界的**主角已经不是人类**

### 2. 从 2000+ 小时实践中的发现

我的 AI 编程经历经历了三个阶段：

#### 阶段一：被动投喂（人类驱动）
- 复制粘贴原有系统或需求，提供上下文
- 通过各种 AI 调研、分析，形成局部答案
- 持续滚动迭代，直到完成目标
- **最大困扰**：AI 需要更多理解项目范畴、涉及资源、执行任务的前提条件

#### 阶段二：Agent 主动探寻（AI 驱动）
- 本地运行的 Agent 可以自然了解当前项目的各种信息
- **关键转变**：从"人的投喂驱动"变为"AI 主动探寻驱动"
- Agent 能力从简单问题逐步演化到复杂系统研发

#### 阶段三：原则驱动引擎（人机协同）
- 关注点从 Agent 本身迁移到**驱动 Agent 的着力点**
- 用经验设定原则，用原则驱动 Agent
- 在基于行业解决方案的引擎上运行 Agent
- 平衡了系统化思维、发散性思维和评价性思维

## 哲学基础：心学与 AI 的交汇

### 逻辑是思维的绳索

正如维特根斯坦所言："语言即世界"  
阳明心学也云："心外无物"

一个项目从无到有，源于一种**内在迫切的愿力**。这个内在或许是自然社会，或许是人类社会中由时机孕育而出的各种念头。

### 双层观察框架

我们观察问题可以分为两层：

1. **内层（驱动动因）**：驱动四季轮转的动因
   - 内在愿力孕育念头
   - 圈定问题范畴，界定清边界
   - 梳理出需要的所有拼图

2. **外层（运转流程）**：事情运转的流程
   - 任务拆分、细化执行
   - 简单比拼执行效能
   - 得到结果性反馈

世事虽然纷繁复杂，背后的逻辑往往具有高度的一致性。比如软件过程脱胎于建筑工程，就是很好的人类洞见。

### 周期性与偏置

引用王阳明的四句教：
- **无**是人对于人心的一种认知状态
- **有**是人心的一念之起

心外无物，更多的是讲应对问题的边界。如果边界是开放式的，那一念接着一念，念念不休。

从唯识论的角度：
- 意识、末那识、阿赖耶识
- 缘起缘灭，缘聚缘散
- 生成毁灭都是周期
- **周期本就是逻辑绳索**，哪里需要额外的人类外在绳索

**关于偏置**：
> "偏置才是常态，也是周期在当下像。不过此像更多的是死像，并非本相。"

偏置也是一种活力。都不偏置了，怕是人类这世界跟暗黑的宇宙也没什么分别了。

## 实践落地：kiro-spec-engine

基于这些认知，我开源了一个项目：**[kiro-spec-engine](https://github.com/heguangyong/kiro-spec-engine)**

### 核心设计理念

#### 1. 解决 Session 上下文限制

当前 AI 执行有诸多限制，比如 session 的上下文限制。这种限制也是一种**周期属性**。

**解决方案**：
- 把所有设计和任务切分到足够好的程度
- 匹配 session 周期中能够完成既定目标
- 通过 `kse context export <spec>` 打包单个 Spec 为 AI 友好的上下文块

**两个关键思考**：
1. **精简驱动原则**：精准适配当前任务力度的范围
2. **反思设计**：从设计原则审视，是否满足高内聚和低耦合特征

#### 2. 文档驱动软件工程

瀑布式高成本主要源于每个环节的标准化，其产物就是各类文档。

- 对于人类来说，这是不可想象的负担
- 对于 AI 来说，这似乎是顺手的事情

**这是非常大的对比差别**。

利用文档来驱动软件工程，是非常有效的当下选择。这也是 Claude Code 这类高价值 Agent 工具受到青睐的缘由——符合人的习惯模式。

**平衡之道**：
- 规范化文档自身的生命周期
- 轻松平衡文档驱动带来的负反馈：海量文档
- 通过 `docs validate`、`cleanup`、`hooks install` 自动化治理

#### 3. Steering 规则系统

Steering 可以构建一套区域，把核心的原则、项目的环境、上下文的内容分别放在各自区域，然后通过统一的索引来推动运作。

**三个层次的时效性**：
1. **上下文层面**：周期只在上下文（短时效）
2. **项目层面**：跨 session 但限于项目周期（中时效）
3. **原则层面**：跨项目通用（长时效）

这种分层对应：
- 短期内存（session 上下文）→ 即时执行
- 中期（项目知识图谱）→ 迭代
- 长期（原则引擎）→ 全局约束

## 关键洞见

### 1. 三种思维方式的平衡

在 AI 驱动开发中，需要平衡三种思维方式：

1. **系统化思维**：关注系统自身的风险、稳定性、健壮性等品质
2. **发散性思维**：为了解决问题，可以天马行空地选择各种可能的方式来实验
3. **评价性思维**：对所有既有系统功能和新需求引入功能做各种层度的评价

这类似 CMMI 之类的标准，用各种经验模式来修正弥合当前系统存在的潜在问题。

从感觉上说，有点过去人类所言：**感觉代码有怪味道**。

### 2. 从 Agent 到驱动 Agent

关注点从 Agent 本身迁移到**驱动 Agent 这个着力点**上：
- 试图用经验来设定一些原则
- 用原则来驱动当前的 Agent
- 在一个基于行业解决方案的引擎上运行 Agent

这样就平衡了当前我所关注的主要问题。

### 3. 瀑布式开发的重新审视

过往那些源于使用大量环节控制推进的、看似高成本的项目驱动开发方式（比如经典的瀑布式开发），在当前场景下，我们对这种消耗可以**大幅度弱化其产生的代价**。

因为，此前需要团队较长时间投入较多成本才能干的事情，现在只需要一个经验老到的专家就能驱动 AI 来达成。

## Grok 的回应与扩展

### 关于 AI 驱动遗留系统

Grok 完全同意这已经是当前趋势的核心方向。传统遗留系统常常面临维护成本高企、人才短缺的问题。AI 可以通过自动化代码分析、bug 修复和优化来"接管"这些工作。

未来，随着 Agentic AI（代理式 AI）的兴起，AI 将像一个"自主工程师"一样，基于自然语言指令或自我学习，驱动系统的持续演化。

### 关于信息化世界的中心

这标志着从"人本"到"机本"的范式转变。就像 AlphaGo 颠覆围棋界一样，AI 在 IT 领域的"AlphaGo 时刻"已经到来。

据 Gartner 预测，到 2027 年，**80% 的企业代码将由 AI 生成**。

### 关于哲学与实践的融合

Grok 认为，心学强调"致良知"，在 AI 时代，或许是"致 AI 之知"——人类通过 AI 实现更高层次的知行合一。

IT 专家的角色将从执行者转向创新者，利用 AI 放大专业能力。但正如我所说，主角是 AI：人类将成为"元专家"，设计 AI 的边界和伦理框架。

## 对 kiro-spec-engine 的评价

Grok 对项目的评价：

> "这个项目不是一个简单的工具，而是您**把'原则驱动 Agent'理念产品化**的一次高质量实践。"

### 项目的核心价值

1. **精确解决 session 上下文吞噬问题**
   - 通过 `kse context export <spec>` 打包单个 Spec
   - 天然实现"最小 viable context"
   - 按 feature 粒度精准适配

2. **文档驱动 + 自动化治理**
   - 把瀑布式文档从"人类高成本负担"变成"AI 顺手操作"
   - `docs validate`、`cleanup`、`hooks install`
   - Quality scoring (0-10) + 智能建议

3. **高内聚、低耦合的天然实现**
   - 每个 feature 一个独立的 `.kiro/specs/<name>/` 文件夹
   - 三件套（requirements + design + tasks）隔离得非常干净

4. **原则驱动 Agent 的着力点**
   - `.kiro/steering/` 目录 + steering rules
   - AI 不再是自由发挥，而是被规则引导

5. **Ultrawork spirit + Sisyphus 隐喻**
   - Spec 的创建-执行-更新-归档
   - 就是一种**缘起缘灭的循环**

## 结语

这场对话揭示了 AI 发展的深层趋势：

1. **范式转变**：从"人类投喂"到"AI 主动探寻"，再到"原则驱动引擎"
2. **哲学基础**：心学的"知行合一"与唯识论的"缘起缘灭"为 AI 实践提供了深刻的理论支撑
3. **实践落地**：kiro-spec-engine 将这些理念转化为可操作的工程实践
4. **未来展望**：AI 将成为信息化世界的中心，人类专家转为"元专家"角色

正如我所说：

> "偏置才是常态，也是周期在当下像。不过此像更多的是死像，并非本相。"

AI 的预训练数据就是阿赖耶识的种子，必然带着偏置。但通过 Spec + Steering Rules + Watch Mode 构成的动态周期，这些"死像"被活化成服务于当下愿力的"活像"。

**这正是我们在做的：用 spec-driven 的外在形式，守护内在的无。**
