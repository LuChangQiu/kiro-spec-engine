## 新需求记录

### 自动依赖安装功能
**需求**: 当用户尝试读取某种格式文件但缺少解析器时，kse 自动安装对应的 npm 包

**场景**:
- 读取 YAML 文件缺少 js-yaml → 自动安装
- 读取 TOML 文件缺少 @iarna/toml → 自动安装
- 读取 XML 文件缺少 xml2js → 自动安装

**实现思路**:
1. 创建 DependencyManager
2. 检测缺失的依赖
3. 提示用户并自动安装
4. 重试操作

**优先级**: 中等（增强功能）

## 使用示例

```bash
# 初始化知识库
kse knowledge init

# 添加设计模式
kse knowledge add pattern "Singleton Pattern" \
  --tags "design-pattern,creational" \
  --category architecture

# 添加经验教训
kse knowledge add lesson "Database Migration Pitfalls" \
  --tags "database,migration" \
  --category backend

# 列出所有条目
kse knowledge list

# 按类型过滤
kse knowledge list --type pattern

# 搜索
kse knowledge search "database"

# 查看详情
kse knowledge show kb-xxx

# 统计
kse knowledge stats
```

## 技术亮点

1. **轻量级**: 不自动加载到 AI 上下文，按需查询
2. **模块化**: 6 个独立管理器，职责清晰
3. **可扩展**: 支持自定义模板和类型
4. **用户友好**: 彩色输出，表格展示
5. **数据完整性**: YAML frontmatter + Markdown 内容
